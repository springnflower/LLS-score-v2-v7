# 로그인 화면을 기존 Vercel 주소에 반영하기

**기존 링크:** https://llsscoreboardv3-sd.vercel.app  
**목표:** 이 주소는 그대로 두고, 여기에 “로그인 화면”이 적용된 최신 버전이 보이게 하기.

---

## 1. 전체 흐름 (왜 이렇게 하면 되는지)

```
[지금]  로컬 v7 폴더에 로그인 코드 있음
          ↓
[1단계] 이 코드를 Git으로 “저장” (commit)
          ↓
[2단계] GitHub에 올리기 (push)
          ↓
[3단계] Vercel이 자동으로 “새 코드로 다시 배포”
          ↓
[결과]  같은 주소(llsscoreboardv3-sd.vercel.app)로 접속하면 로그인 화면이 나옴
```

- **새 퍼블리싱이 아님** → 기존 프로젝트에 “한 번 더 배포”하는 것.
- **주소는 그대로** → https://llsscoreboardv3-sd.vercel.app 유지.

---

## 2. 사전 확인 (한 번만 하면 됨)

Vercel에 올라가 있는 사이트가 **어느 GitHub 저장소**와 연결돼 있는지 확인해야 합니다.

1. 브라우저에서 **https://vercel.com** 접속 후 로그인
2. **Dashboard**에서 **llsscoreboardv3-sd** (또는 비슷한 이름) 프로젝트 클릭
3. 상단 메뉴에서 **Settings** → 왼쪽에서 **Git** 클릭
4. **Connected Git Repository**에 나오는 저장소 주소 확인  
   - 예: `github.com/본인아이디/저장소이름`

지금 v7 폴더가 이 **같은 저장소**에 연결돼 있어야, 아래에서 push한 내용이 이 Vercel 프로젝트에 자동 반영됩니다.

v7에서 연결된 저장소 확인:

```bash
cd /Users/myoungsookim/LLS_스코어보드/v7
git remote -v
```

여기 나오는 주소가 Vercel **Git** 설정에 나온 저장소와 같으면 OK.

---

## 3. 로그인 코드를 GitHub에 올리기 (실제로 할 작업)

아래 명령은 **Cursor 터미널** 또는 **Mac 터미널**에서 **v7 폴더로 이동한 뒤** 한 줄씩 실행하면 됩니다.

### 3-1. v7 폴더로 이동

```bash
cd /Users/myoungsookim/LLS_스코어보드/v7
```

### 3-2. 변경된 파일 모두 스테이징

```bash
git add app/layout.tsx components/auth-gate.tsx components/login-form.tsx
```

(또는 모든 변경을 넣으려면 `git add .`)

### 3-3. 커밋 (로컬에 “저장”하는 단계)

```bash
git commit -m "첫 화면 로그인 추가 (ID: springnflower)"
```

### 3-4. GitHub에 올리기 (push)

```bash
git push origin main
```

- `main`이 아니라 `master` 등 다른 브랜치를 쓰고 있다면, 그 이름으로 바꾸세요.  
  - 예: `git push origin master`
- GitHub 비밀번호를 묻는 경우, **비밀번호 대신 Personal Access Token**을 입력해야 할 수 있습니다.  
  - GitHub → **Settings** → **Developer settings** → **Personal access tokens** 에서 토큰 생성 후, 비밀번호 자리에 토큰 입력.

**여기까지 하면:** GitHub 저장소에 “로그인 추가” 버전이 올라간 상태입니다.

---

## 4. Vercel에서 배포 확인

1. **https://vercel.com** → 해당 프로젝트(**llsscoreboardv3-sd** 등) 클릭
2. **Deployments** 탭 클릭
3. 맨 위에 **Building** 또는 **Ready** 상태의 새 배포가 생겼는지 확인  
   - 방금 push한 직후라면 1~2분 안에 **Building** → **Ready**로 바뀝니다.
4. **Ready**가 되면 **Visit** 또는 브라우저에서 **https://llsscoreboardv3-sd.vercel.app** 접속
5. **첫 화면에 로그인 화면**이 나오고,  
   - ID: `springnflower`  
   - PW: `llsstudio`  
   로 로그인 가능한지 확인하면 됩니다.

---

## 5. 문제가 생겼을 때

| 상황 | 확인할 것 |
|------|------------|
| push가 안 됨 (권한 오류) | GitHub 로그인, Personal Access Token 사용 여부 |
| Vercel에 새 배포가 안 뜸 | Vercel **Settings → Git** 에서 연결된 저장소·브랜치가 push한 저장소/브랜치와 같은지 확인 |
| 배포는 됐는데 로그인 화면이 안 나옴 | 배포가 **Ready**까지 끝났는지, 브라우저 **새로고침** 또는 시크릿 창으로 접속해서 확인 |

## 요약

- **기존 링크에 적용 가능합니다.** 새로 퍼블리싱할 필요 없이, 같은 URL에 “한 번 더 배포”하면 됩니다.
- **할 일:** v7에서 `git add` → `git commit` → `git push origin main` 까지 하면, Vercel이 자동으로 그 주소를 최신 버전(로그인 포함)으로 갱신합니다.
